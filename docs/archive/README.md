# 归档文档

本目录存放项目开发过程中的历史文档，这些文档记录了项目的演进过程，但不是日常使用所需的核心文档。

## 📚 文档说明

### 开发过程记录
- **P0_COMPLETION_CHECKLIST.md** - P0阶段任务完成清单（2026-07-03完成）
- **P1_COMPLETION_REPORT.md** - P1阶段完成报告，记录了反偏见工具等功能的实现

### 设计与规划
- **OPTIMIZATION_PLAN.md** - 借鉴 ai-berkshire 的优化方案记录（大部分已实施）

### 操作指南
- **NEXT_STEPS.md** - 发布前的操作步骤指南
- **PUBLISH.md** - skills.sh 发布指南
- **MIGRATION_GUIDE.md** - 从旧版本迁移到新版本的指南

## 🔍 为什么归档？

这些文档在开发阶段非常重要，但项目发布后：
- **P0/P1报告**：任务已完成，功能已合并到主分支
- **NEXT_STEPS/PUBLISH**：发布步骤已执行完毕
- **OPTIMIZATION_PLAN**：优化已实施，保留作为设计决策的历史记录
- **MIGRATION_GUIDE**：随着项目成熟，迁移需求减少

## 📖 如何查看？

如需查阅这些历史文档，可以：
1. 直接在本目录浏览
2. 通过 Git 历史查看文档演进过程
3. 相关功能的最新说明请参考根目录的核心文档
